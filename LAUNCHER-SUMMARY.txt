================================================================================
                              TROUBLESHOOTING
================================================================================

MySQL Connection Error?
  → Ensure MySQL is running: mysql -u root -p
  → Check credentials in launcher script
  → Verify user 'tradevault' exists

Port Already in Use?
  → Launcher will ask to kill existing process
  → Or manually: kill $(lsof -t -i :3001)

npm Install Takes Too Long?
  → Normal on first run (downloads ~500MB dependencies)
  → Subsequent runs are much faster
  → Check internet connection if stuck >5 minutes

Frontend Not Loading?
  → Wait 5-10 seconds for Vite to compile
  → Check console errors in Frontend window
  → View logs: tail -f /tmp/tradelab-frontend.log

See LAUNCHER-README.md for complete troubleshooting guide.

================================================================================
                        WHAT THE SCRIPTS DO IN DETAIL
================================================================================

Step 1: Prerequisites Check
  - Verifies Node.js is installed and in PATH
  - Verifies npm is installed and in PATH
  - Verifies MySQL is installed and in PATH
  - Displays all installed versions
  - Exits if any prerequisite is missing

Step 2: Create Screenshots Directory
  - Creates ~/TradeVault/screenshots/ if it doesn't exist
  - Sets appropriate permissions
  - Used for storing all uploaded trade screenshots
  - Auto-organized by year/month/day/HHMMSS

Step 3: Start MySQL Service
  - Detects operating system (macOS, Linux, Windows)
  - Uses appropriate method to start MySQL:
    * macOS: brew services start mysql
    * Linux: sudo systemctl start mysql
    * Windows: net start MySQL80
  - Attempts connection with configured credentials
  - Verifies database 'tradevault' is accessible
  - Exits if connection fails

Step 4: Start Backend Server
  - Changes directory to server/
  - Runs npm install (silently) to ensure dependencies
  - Starts Node.js Express server with 'npm start'
  - Waits 3 seconds for server to initialize
  - Makes HTTP request to http://localhost:3001/api/health
  - Logs output to /tmp/tradelab-backend.log
  - Displays process ID for manual management
  - Returns to project root directory

Step 5: Start Frontend Dev Server
  - Installs npm dependencies (if needed)
  - Starts Vite dev server with 'npm run dev'
  - Waits 5 seconds for compilation
  - Makes HTTP request to http://localhost:8081
  - Logs output to /tmp/tradelab-frontend.log
  - Displays process ID for manual management

Cleanup (on Ctrl+C):
  - Captures the SIGINT signal (Ctrl+C)
  - Kills backend Node process using PID
  - Kills frontend Node process using PID
  - Displays confirmation messages
  - Exits gracefully
